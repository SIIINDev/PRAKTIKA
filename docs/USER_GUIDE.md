# Руководство пользователя

Интеллектуальная поисковая система по внутренней базе знаний университета.
Документ описывает работу с веб-приложением: загрузку документов и поиск по ним.

> Это требование **QA-06** («Руководство пользователя»).

---

## 1. Что умеет система

- Загружать документы в форматах **PDF** и **DOCX** (до 20 МБ каждый).
- Автоматически извлекать из них текст и строить поисковый индекс.
- Выполнять полнотекстовый поиск по всем загруженным документам.
- Показывать найденные фрагменты с подсветкой совпадений и номером страницы.
- Ранжировать результаты по релевантности и листать их по страницам.

---

## 2. Как открыть приложение

1. Откройте браузер.
2. Перейдите по адресу **http://localhost:8080** (или адресу, который сообщил администратор).
3. Вверху страницы есть две вкладки: **Документы** и **Поиск**.

---

## 3. Загрузка документов

1. Откройте вкладку **Документы**.
2. В блоке «Загрузка документов» есть область для перетаскивания файлов.
3. Загрузить файлы можно двумя способами:
   - **Перетащить** один или несколько файлов мышью прямо в эту область («Перетащите файлы сюда»);
   - **Нажать** на область — откроется системное окно выбора файлов; можно выбрать сразу несколько.
4. Поддерживаются только файлы **PDF** и **DOCX** размером **до 20 МБ**. Файлы других форматов
   будут отклонены.

### Статусы обработки

После добавления файла вы увидите его в очереди загрузки с индикатором прогресса и статусом:

| Статус | Что означает |
|--------|--------------|
| **Загрузка…** | Файл передаётся на сервер (показывается процент) |
| **Индексация…** | Файл принят; система извлекает текст и строит индекс |
| **Готово** | Документ проиндексирован и доступен для поиска |
| **Ошибка** | Файл не удалось обработать (рядом будет текст причины) |

Дождитесь статуса **Готово** — после этого документ можно искать. Завершённые элементы очереди
можно убрать кнопкой **Скрыть**.

### Список загруженных документов

Справа отображается список всех загруженных документов: имя файла, дата загрузки, размер,
число извлечённых фрагментов и статус. Кнопка **Обновить** перечитывает список.
Кнопка **Удалить** удаляет документ вместе с его данными в поисковом индексе (потребуется
подтверждение — действие необратимо).

---

## 4. Поиск по базе знаний

1. Откройте вкладку **Поиск**.
2. Введите запрос в поле ввода (например, «нейронные сети» или «база данных»).
3. Запустите поиск:
   - нажмите кнопку **Найти**, или
   - нажмите клавишу **Enter**.
4. Под полем ввода показываются недавние запросы в виде кнопок-«чипов» — нажатие на чип
   повторяет соответствующий поиск.

### Как читать результаты

Каждый результат — это карточка, в которой есть:

- **Имя файла** и **номер страницы** («стр. N»), где найден фрагмент;
- **Фрагмент текста** с найденным контекстом;
- **Подсветка** — слова, совпавшие с запросом, выделены **жёлтым фоном**;
- **Оценка релевантности** (число справа вверху) — чем выше, тем точнее совпадение.

Над списком отображается сводка: сколько результатов найдено, сколько миллисекунд занял
запрос, и метка **из кэша**, если идентичный запрос уже выполнялся недавно (ответ отдан
из кэша быстрее).

### Пагинация (постраничная навигация)

Результаты выводятся по **10 на страницу**. Внизу списка появляются кнопки страниц и
стрелки «‹» / «›». При смене страницы список прокручивается к началу.

### Если ничего не найдено

Если по запросу нет совпадений, появится сообщение:

> «По вашему запросу ничего не найдено. Попробуйте изменить формулировку»

Попробуйте:
- переформулировать запрос или использовать синонимы;
- сократить запрос до ключевых слов;
- проверить, что нужный документ загружен и имеет статус **Готово**.

## 5. Устранение неполадок

| Проблема | Возможная причина | Что сделать |
|----------|-------------------|-------------|
| Файл не загружается, сразу «Ошибка» | Неподдерживаемый формат (не PDF/DOCX) или размер > 20 МБ | Используйте PDF или DOCX до 20 МБ |
| Статус «Ошибка» после индексации | Файл пустой, повреждён или не содержит извлекаемого текста (например, скан без распознавания) | Проверьте файл; для сканов нужен текстовый слой (OCR) |
| Документ загружен, но не находится в поиске | Индексация ещё не завершена | Дождитесь статуса **Готово** и повторите поиск |
| Поиск возвращает «ничего не найдено» для слов из документа | Слишком специфичная формулировка | Упростите запрос, используйте ключевые слова |
| Страница не открывается | Стек не запущен или занят порт | Убедитесь, что выполнено `docker compose up`; проверьте порт `FRONT_PORT` в `.env` |
| Поиск отвечает ошибкой сервиса | Поисковый движок недоступен | Проверьте состояние системы: http://localhost:8000/health — все зависимости должны быть `up` |

### Проверка состояния системы

Откройте http://localhost:8000/health. Ответ вида
`{"status":"ok","elasticsearch":"up","redis":"up","postgres":"up"}` означает, что все
компоненты работают. Если какой-то компонент `down`, обратитесь к администратору.

## 6. Частые вопросы

**Сколько документов можно загрузить?** Ограничения по количеству нет; ограничен только
размер одного файла (20 МБ).

**Поддерживается ли русский язык?** Да, индекс настроен на русскоязычный анализатор —
поиск учитывает разные формы слов (склонения, числа).

**Почему результат отдаётся «из кэша»?** Идентичные запросы кэшируются на 5 минут, чтобы
ускорить повторный поиск. По истечении срока запрос снова выполняется в индексе.

**Можно ли удалить документ?** Да, кнопкой **Удалить** в списке документов; документ и его
данные в индексе удаляются безвозвратно.
